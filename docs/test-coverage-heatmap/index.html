갭 분석 — 남은 2개 항목 + 분류

🔴 연기됨 — PTY 또는 실제 터미널 필요

runtime::run

crossterm raw mode + alternate screen + mouse capture로 stdout을 점유합니다. TestBackend만으로는 검증 불가. expectrl 기반 PTY 테스트 또는 run 시그니처를 &mut Terminal<B>로 리팩터링하여 Backend 주입이 필요.

runtime::App trait

init / on_event / should_quit 메서드가 run을 통해서만 의미를 가짐. run이 테스트 가능해지면 자동으로 커버.

⚪ 수용 가능 — 테스트 가치 대비 비용 낮음

Default impls (Spinner, KeyHelp, StyleBuilder, TextInput, Command)

단순 new() 위임. 생성자가 이미 테스트됨.

private helpers (Column::constraint, runtime::main_loop/setup/restore)

이미 통합 테스트 된 public API의 내부 구현. 직접 단위 테스트는 중복.

drain_messages 추가 엣지케이스

현재 None/Msg/Batch(중첩 포함) 커버. 추가 케이스는 수익 감소.
